静态代理

特点：静态代理在编译时就已经实现，编译完成后代理类是一个实际的class文件

静态代理demo演示

创建一个代理类。在调用源方法的过程中，假如自己的代码。

缺点显而易见。每一个类的代理都需要增加这样一个代理类。不方便维护。假设在每个类的每个方法中都想加入某一个日志功能。在不改变源码、使用静态代理的情况下，需要为每一个类创建一个代理类，然后重写每一个方法。非常麻烦。

动态代理(jdk代理)

特点：动态代理对象不需要实现接口，但是要求目标对象必须实现接口。动态代理是在运行时动态生成的，即编译完成后没有实际的class文件，而是在运行时动态生成类字节码，并加载到JVM中

使用 java.lang.reflect.Proxy 类来进行动态代理。

因为传进去中有个参数是 target.getClass().getInterfaces() 所以也能看出能被代理的只能是接口中的方法。

Proxy 生成一个代理类实现了所有Interface中的方法。（实现的结果就是，每一个方法的调用全部变成调用InvocationHandler）。这个可以通过反编译动态代理类的源码得出

核心流程:

  • 创建了一个接口的克隆类

  • 通过getProxyClass0方法动态生成Class类

    以下是动态生成Class类流程：

    • getProxyClass0->

    • proxyClassCache.get(loader, interfaces)->

    • V value = supplier.get()->

    • value = Objects.requireNonNull(valueFactory.apply(key, parameter))->

    • valueFactory.apply(key, parameter)

    定义代理类的名称proxyName,然后通过PrxoyGenerator.generateProxyClass生成字节码文件

  • 通过defineClass0方法去加载这个类

  • 最后再实例化

Cglib代理

特点：运行时在内存中动态生成一个子类对象从而实现对目标对象功能的扩展。但是无法对final class final method 进行代理

Cglib在使用过程中需要注意一点：

Cglib的方法调用不是通过反射来完成的。而是通过方法索引来调用的。（jdk动态代理使用反射来完成调用 method.invoke）

Cglib 方法索引调用示例**（FastClass实现机制：FastClass其实就是对Class对象进行特殊处理，提出下标概念index，通过索引保存方法的引用信息，将原先的反射调用，转化为方法的直接调用）:**

可以通过生成cglib代理的class源码来学习。
